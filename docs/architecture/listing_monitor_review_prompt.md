## 📋 Review范围和输出结构

### Part 1: 项目全景理解（Executive Summary）

请先快速浏览整个codebase，然后用**500字以内**回答：

1. **核心价值主张**：这个产品解决什么问题？目标用户是谁？
2. **技术栈和架构**：主要用了什么技术？整体架构是什么模式（ETL? Event-driven? Batch pipeline?）
3. **数据流向**：从raw data到最终用户看到的insights，经历了哪几层转换？
4. **当前成熟度**：如果0=demo, 5=MVP, 10=production-ready，你会打几分？为什么？

### Part 2: 代码质量深度分析（Code Quality Deep Dive）

针对以下维度给出评分（1-10分）和具体建议：

#### 2.1 代码组织与可维护性
- 文件/目录结构是否合理？
- 模块划分是否清晰？（data fetching, transformation, storage, UI是否解耦？）
- 命名规范如何？
- 有没有明显的代码重复？
- 配置管理做得怎么样？（硬编码 vs 配置文件）

**输出格式**：
```
评分：X/10
优点：[列出2-3个做得好的地方]
问题：[列出top 3最需要改进的问题，按优先级排序]
Quick Win：[1-2个可以立即改进且impact大的点]
```

#### 2.2 数据质量和健壮性
- Error handling是否充分？（API失败、数据缺失、格式异常）
- 有没有数据验证逻辑？
- Retry和rate limiting做得如何？
- 日志记录是否完善？
- 有没有data quality monitoring？

#### 2.3 性能和可扩展性
- 当前能处理多少数据量？（多少交易所、多少币种）
- 如果数据量10x增长会在哪里遇到瓶颈？
- API调用是否高效？（有无并发、缓存、batch处理）
- 数据库设计是否合理？（如果用了的话）

#### 2.4 测试和文档
- 有测试吗？覆盖率如何？
- README是否清晰？新人能否根据文档跑起来？
- 代码注释质量如何？
- 有没有data schema文档？

### Part 3: 架构和设计模式（Architecture Review）

#### 3.1 当前架构图
请用**ASCII art或文字描述**画出当前的架构图，包括：
- 数据源
- 处理层（fetch, clean, enrich, analyze）
- 存储层
- 展示层
- 定时任务/触发机制

#### 3.2 架构优缺点
- 当前设计的最大优点是什么？
- 最大的技术债是什么？
- 如果重构，你会改哪3个地方？（按ROI排序）

#### 3.3 可扩展性建议
针对以下场景，给出具体的扩展方案：
1. **新增数据源**：如果要加5个新交易所（Kraken, Coinbase, KuCoin等），需要改多少代码？
2. **新增feature**：如果要加"RWA币种分类"功能（这是我当前要做的），应该插入到哪个模块？
3. **实时监控**：如果要从"每日batch"升级到"实时推送新上币通知"，架构需要怎么改？

### Part 4: 产品层面建议（Product Perspective）

#### 4.1 功能完整性
- 当前feature set是否足够解决目标用户的问题？
- 有哪些obvious missing features？
- 如果只能加1个新功能，你会加什么？为什么？

#### 4.2 用户体验（如果有UI的话）
- Streamlit界面的设计如何？
- 数据展示是否清晰？
- 有没有让用户"aha moment"的功能？

#### 4.3 商业价值和差异化
- 这个项目和市面上类似工具（如Coinmarketcap的listing tracker）相比，差异化在哪？
- 如果要向投资人/老板pitch这个项目，你会怎么讲故事？
- 有哪些潜在的monetization机会？

### Part 5: 行动清单（Actionable Recommendations）

请给出一个**优先级排序的改进清单**，分为3个层级：

#### P0 - 必须修复（Critical）
> 影响正确性、安全性、或导致系统不稳定的问题

示例格式：
```
1. [问题描述] 
   影响：[具体说明会导致什么后果]
   解决方案：[具体的修复步骤，最好有代码示例]
   预计工作量：[小时数]
```

#### P1 - 强烈建议（High Impact）
> 能显著提升代码质量、性能、或用户体验的改进

#### P2 - Nice to Have（Enhancement）
> 锦上添花的优化

### Part 6: 技术亮点和可复用模式（Highlights）

请指出：
1. 这个项目中做得特别好、值得保留和推广的设计/代码
2. 有哪些模式可以抽象成library或template供其他项目使用
3. 如果用这个项目去面试，应该重点讲哪3个技术点？
